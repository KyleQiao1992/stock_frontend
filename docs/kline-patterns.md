# K线形态识别 —— 书面定义 ↔ 代码实现 对照表

> 这份文档是「书里的形态定义」和「代码里的判定条件」一一对照的核查表，方便逐条核对。
> 每次新增/修改一个形态，都先把书面"形态特征"原文抄进来，再写出对应的代码条件和状态。

## 0. 代码位置

- 识别函数：`detectKlinePatterns(rows)` —— `src/components/AShareTD9InteractiveChart.jsx`
- 标记渲染：同文件 `Chart` 组件内，`showPatterns &&` 的那段 SVG（详见 §6）
- 开关按钮：顶部工具栏「K线形态」（`ChartToolbar`）

返回结构：`[{ index, name, abbr, dir }]`，`dir` 为 `"bear"`（顶部看跌）/ `"bull"`（底部看涨）。
同一根K线若命中多个形态，按 `priority`（越小越具体/强）只保留一个。

## 1. K线数据与几何量

每根K线是 `{ date, open, close, high, low, volume }`。几何量由 `feat(d)` 计算：

| 量 | 定义 | 含义 |
|---|---|---|
| `body` | `|close - open|` | 实体长度 |
| `range` | `high - low` | 振幅（整根高度）|
| `upper` | `high - max(open, close)` | 上影线 |
| `lower` | `min(open, close) - low` | 下影线 |
| `up` / `down` | `close > open` / `close < open` | 阳线 / 阴线 |

## 2. 趋势判定 `trend(endIdx, lookback = 4)`

用来判断"上升波段 / 下跌波段"。比较 `endIdx` 与往前 `lookback` 根的收盘价：

```
chg = (close[endIdx] - close[endIdx-4]) / |close[endIdx-4]|
chg >  1%  → 1  （上行）
chg < -1%  → -1 （下行）
否则       → 0
```

单根反转形态判趋势时用 `trend(i - 1)`，即看**该形态前一根之前的 4 根**走势，不含形态本身。

## 3. 可调旋钮（书里**没有**给数值、由实现量化的部分）

书面定义里用了定性词（"附近""很短""上升波段"），代码必须落成数字。**以下是仅有的几个我定的旋钮**，其余条件全部按书面原话硬编码：

| 书面措辞 | 代码取值 | 说明 |
|---|---|---|
| 开盘价在前一根最高价**附近** | `open ≥ prev.high × 0.99` | 容差 1%，允许略低于最高价或更高 |
| **很短**的上/下影线 | `影线 ≤ range × 0.15` | 占振幅 15% 以内 |
| 黄昏之星**小实体/小影线** | `body/upper/lower ≤ prev.body × 0.35` | 与前一根实体比较，表示高位小星 |
| 黄昏之星影线**相差不大** | `|upper-lower| ≤ prev.body × 0.15` | 上下影线长度接近 |
| 长腿车夫**小实体** | `body ≤ range × 0.15` | 允许近似十字，但要求整根K线有足够长影 |
| 长腿车夫**长影线** | `upper/lower ≥ max(body × 2, range × 0.3)` | 既满足"2倍实体"，也避免普通小十字误标 |
| 墓碑线**小实体/长上影** | `body ≤ range × 0.14`、`upper ≥ range × 0.62` | 按当前代码的高位墓碑十字形态 |
| 墓碑线**很短下影** | `lower ≤ range × 0.1` | 无下影线或只有很短下影线 |
| 中/长实体 | `body ≥ range × 0.4` | 用于乌云盖顶的中阳/长阳、中阴/长阴 |
| 收在最低价附近 | `lower ≤ range × 0.15` | 用于乌云盖顶第二根阴线 |
| 中小阴线 | `down && range×0.25 ≤ body ≤ range×0.75` | 用于三只乌鸦，排除十字和超长实体 |
| **上升/下跌波段** | `trend`：4 根、收盘涨跌幅 ±1% | 见 §2 |

> 影线"≥实体2倍"、"前一根最高价"这类**有明确数值/对象**的条件，按书面原话写死，不在旋钮内。

## 4. 形态逐条对照

### 4.1 射击之星 ✅ 已核对（顶部反转 / 看跌，标记「射」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价在前一根K线的最高价附近；
2. 上影线长度是实体的2倍以上；
3. 没有下影线或只有很短的下影线。

> 颜色不重要（绿阴更有效）；实体越短越看跌；通常伴随放量。

**代码条件**（全部满足才标记）：
```js
i >= 1 && f.body > 0 &&
f.upper >= f.body * 2 &&          // ② 上影 ≥ 实体2倍
f.lower <= f.range * 0.15 &&      // ③ 无/很短下影
c.open  >= rows[i-1].high * 0.99 && // ①b 开盘在前一根最高价附近
trend(i - 1) > 0                  // ①a 上升波段顶部
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘价在前一根最高价附近 | `open ≥ prev.high × 0.99` | ✅ |
| ② 上影线 ≥ 实体2倍 | `upper ≥ body × 2` | ✅ |
| ③ 无/很短下影 | `lower ≤ range × 0.15` | ✅ |
| 颜色不限、实体越短越看跌 | `body > 0`，不设实体下限 | ✅ |

**取舍说明：** 书面①b（开盘≈前一根最高价）与书中**图3-2实例**（"以前日收盘价开盘"）相互冲突。本实现**忠于书面文字**，因此图3-2那种"开在前收"的射击之星不会被标。

### 4.2 上吊线 ✅ 已核对（顶部反转 / 看跌，标记「吊」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价在前一根K线最高价附近；
2. 没有上影线或只有很短的上影线；
3. 下影线长度是实体的2倍以上；
4. 小的实体，实体颜色不很重要。

> 收阴线（绿实体）表明收盘不能反弹至开盘价，卖方动能在聚集。

**代码条件：**
```js
// 共同形状（②③④）
f.body > 0 && f.lower >= f.body * 2 && f.upper <= f.range * 0.15
// 上吊线（在上述形状基础上）
&& i >= 1 && trend(i - 1) > 0 && c.open >= rows[i-1].high * 0.99
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘价在前一根最高价附近 | `open ≥ prev.high × 0.99` | ✅ |
| ② 无/很短上影 | `upper ≤ range × 0.15` | ✅ |
| ③ 下影线 ≥ 实体2倍 | `lower ≥ body × 2` | ✅ |
| ④ 小的实体、颜色不重要 | 见下 ⚠️ 隐式满足 | ⚠️ |

**⚠️ 特征④"小实体"是隐式满足，没有独立显式门槛。** 由 ②③ 推出：实体 ≤ 振幅约 **28%**、且永远 ≤ 下影的一半。若需把④作为独立、更紧的条件（如实体 ≤ 振幅 20%），需另加一行。

### 4.3 锤子线 ⚠️ 形状已定，①b 待核对（底部反转 / 看涨，标记「锤」）

与上吊线**形状完全相同**（长下影、小实体、短上影），靠趋势区分：处**下跌**末端（`trend(i-1) < 0`）即判为锤子线。
书面"开盘位置"等特征**尚未单独核对**，底部形态的开盘规则与顶部不同（通常低开），留待后续。

### 4.4 黄昏之星 ✅ 已核对（顶部反转 / 看跌，标记「昏」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价通常在前日收盘价之上；
2. 上影线与下影线较短，且长度相差不大；
3. 实体很小或开盘价等于收盘价；
4. 实体颜色不重要。

**代码条件**（全部满足才标记，标在小星本身这根K线上）：
```js
i >= 1 &&
prev.body > 0 &&
trend(i - 1) > 0 &&
open >= prev.close &&
body <= prev.body * 0.35 &&
upper <= prev.body * 0.35 &&
lower <= prev.body * 0.35 &&
Math.abs(upper - lower) <= prev.body * 0.15
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘通常在前日收盘价之上 | `open ≥ prev.close` | ✅ |
| ② 上下影线较短 | `upper/lower ≤ prev.body × 0.35` | ✅ |
| ② 上下影线长度相差不大 | `|upper-lower| ≤ prev.body × 0.15` | ✅ |
| ③ 实体很小或开收相等 | `body ≤ prev.body × 0.35`，允许 `body = 0` | ✅ |
| ④ 实体颜色不重要 | 不要求阳线/阴线 | ✅ |

**取舍说明：** 旧实现是"阳线 → 小星 → 阴线跌破首根实体中点"的三根确认版。现已改为书面定义中的**单根小星本体**，确认阴线不参与是否标记。

### 4.5 长腿车夫 ✅ 已核对（顶部反转 / 看跌，标记「车」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 上影线与下影线较长，均长于实体的2倍；
3. 实体较小，实体颜色不重要。

**代码条件**（全部满足才标记）：
```js
i >= 1 &&
trend(i - 1) > 0 &&
body <= range * 0.15 &&
upper >= Math.max(body * 2, range * 0.3) &&
lower >= Math.max(body * 2, range * 0.3)
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 上影线 ≥ 实体2倍 | `upper ≥ body × 2` | ✅ |
| ② 下影线 ≥ 实体2倍 | `lower ≥ body × 2` | ✅ |
| ② 上下影线都很长 | `upper/lower ≥ range × 0.3` | ✅ |
| ③ 实体较小、颜色不重要 | `body ≤ range × 0.15`，不要求阴阳 | ✅ |

**取舍说明：** 仅用"影线 ≥ 实体2倍"会让近似十字的小波动也命中，因此额外要求上下影线各自至少占整根振幅 30%，确保它确实是长腿形态。

### 4.6 墓碑线 ✅ 已核对（顶部反转 / 看跌，标记「墓」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 实体很小或接近十字；
3. 上影线很长，占整根K线振幅的大部分；
4. 没有下影线或只有很短的下影线，实体颜色不重要。

**代码条件**（全部满足才标记）：
```js
i >= 1 &&
trend(i - 1) > 0 &&
body <= range * 0.14 &&
upper >= range * 0.62 &&
lower <= range * 0.1
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 很长上影线 | `upper ≥ range × 0.62` | ✅ |
| ③ 无/很短下影线 | `lower ≤ range × 0.1` | ✅ |
| ④ 很小实体、颜色不重要 | `body ≤ range × 0.14`，不要求阴阳 | ✅ |

**取舍说明：** 当前代码采用的是高位墓碑十字形态（小实体、长上影、短下影）。如果要改为你图中另一种"接近涨停高开后长阴回落"版本，需要单独调整代码口径。

### 4.7 乌云盖顶 ✅ 已核对（顶部反转 / 看跌，标记「乌」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 第一根K线为一根中阳或长阳线；
3. 第二根K线为高开，并收盘在最低价附近的中阴或长阴线；
4. 第二根K线的收盘价在第一根K线实体的半分位以下。

**代码条件**（全部满足才标记）：
```js
i >= 1 &&
prev.up &&
prev.body >= prev.range * 0.4 &&
down &&
body >= range * 0.4 &&
lower <= range * 0.15 &&
trend(i - 1) > 0 &&
open > prev.close &&
close < (prev.open + prev.close) / 2 &&
close > prev.open
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 第一根中阳/长阳 | `prev.up && prev.body ≥ prev.range × 0.4` | ✅ |
| ③ 第二根高开 | `open > prev.close` | ✅ |
| ③ 第二根中阴/长阴 | `down && body ≥ range × 0.4` | ✅ |
| ③ 第二根收在最低价附近 | `lower ≤ range × 0.15` | ✅ |
| ④ 第二根收盘低于前实体半分位 | `close < (prev.open + prev.close)/2` | ✅ |

**取舍说明：** 代码保留 `close > prev.open`，即第二根没有完全吞没第一根；如果收在第一根开盘价以下，会优先归类为看跌吞没。

### 4.8 看跌吞没 ✅ 已核对（顶部反转 / 看跌，标记「吞」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 第一根K线为一根中阳或长阳线；

down &&
body >= range * 0.4 &&
lower <= range * 0.15 &&

| ② 第一根中阳/长阳 | `prev.up && prev.body ≥ prev.range × 0.4` | ✅ |
| ③ 第二根高开 | `open > prev.close` | ✅ |
| ③ 第二根长阴线 | `down && body ≥ range × 0.4` | ✅ |
| ③ 第二根收在最低价附近 | `lower ≤ range × 0.15` | ✅ |

### 4.9 三只乌鸦 ✅ 已核对（顶部反转 / 看跌，标记「鸦」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 第一根K线为高开的中小阴线；
3. 三根K线分别为收盘价依次降低的中小阴线。

**代码条件**（全部满足才标记，标在第三根阴线上）：
```js
i >= 3 &&
trend(i - 3) > 0 &&
first.open > rows[i - 3].close &&
first/second/third are all medium-small down candles &&
second.close < first.close &&
third.close < second.close

// medium-small down candle:
down && body >= range * 0.25 && body <= range * 0.75
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-3) > 0` | ✅ |
| ② 第一根高开 | `first.open > rows[i-3].close` | ✅ |
| ② 第一根中小阴线 | `first.down && 0.25≤body/range≤0.75` | ✅ |
| ③ 三根均为中小阴线 | 三根都满足 `down && 0.25≤body/range≤0.75` | ✅ |
| ③ 收盘价依次降低 | `second.close < first.close && third.close < second.close` | ✅ |

**取舍说明：** "中小阴线"是书面定性词，当前量化为实体占整根振幅 25%-75%。如果需要更宽或更窄，可以单独调这个阈值。

## 5. 标记渲染规则

- **看跌（bear）**：绿色，标在该K线**最高价上方**。
- **看涨（bull）**：红色，标在该K线**最低价下方**。
- 标签为 1 个汉字缩写（射/吊/昏/车/墓/乌/吞/鸦/锤），鼠标悬停显示完整形态名 + 日期。
- 默认半透明，悬停高亮。

## 6. 未纳入硬判定的部分（属"确认/强度"修饰，不是形态特征本身）

书里这些是**确认信号**或**有效性修饰**，当前**不参与**是否标记的判断：

- **确认信号**（需下一根K线）：下一根开盘在形态实体下方 / 下一根阴线收盘低于形态收盘价。
- **放量**：成交量越大，冲顶/见顶概率越高。
- **颜色权重**：绿色（阴线）的射击之星/上吊线更有效。

如需把"确认信号"做成更严格的"已确认"模式，或把放量/颜色作为强度加权，再单独议。
