# K线形态识别 —— 书面定义 ↔ 代码实现 对照表

> 这份文档是「书里的形态定义」和「代码里的判定条件」一一对照的核查表，方便逐条核对。
> 每次新增/修改一个形态，都先把书面"形态特征"原文抄进来，再写出对应的代码条件和状态。

## 0. 代码位置

- 识别函数：`detectKlinePatterns(rows)` —— `src/components/AShareTD9InteractiveChart.jsx`
- 标记渲染：同文件 `Chart` 组件内，`showPatterns &&` 的那段 SVG（详见 §6）
- 开关按钮：顶部工具栏「K线形态」（`ChartToolbar`）

返回结构：`[{ index, name, abbr, dir }]`，`dir` 为 `"bear"`（顶部看跌）/ `"bull"`（底部看涨）。
同一根K线若命中多个形态，按 `priority`（越小越具体/强）只保留一个。

## 1. K线数据与几何量

每根K线是 `{ date, open, close, high, low, volume }`。几何量由 `feat(d)` 计算：

| 量 | 定义 | 含义 |
|---|---|---|
| `body` | `|close - open|` | 实体长度 |
| `range` | `high - low` | 振幅（整根高度）|
| `upper` | `high - max(open, close)` | 上影线 |
| `lower` | `min(open, close) - low` | 下影线 |
| `up` / `down` | `close > open` / `close < open` | 阳线 / 阴线 |

## 2. 趋势判定 `trend(endIdx, lookback = 4)`

用来判断"上升波段 / 下跌波段"。比较 `endIdx` 与往前 `lookback` 根的收盘价：

```
chg = (close[endIdx] - close[endIdx-4]) / |close[endIdx-4]|
chg >  1%  → 1  （上行）
chg < -1%  → -1 （下行）
否则       → 0
```

单根反转形态判趋势时用 `trend(i - 1)`，即看**该形态前一根之前的 4 根**走势，不含形态本身。

## 3. 可调旋钮（书里**没有**给数值、由实现量化的部分）

书面定义里用了定性词（"附近""很短""上升波段"），代码必须落成数字。**以下是仅有的几个我定的旋钮**，其余条件全部按书面原话硬编码：

| 书面措辞 | 代码取值 | 说明 |
|---|---|---|
| 开盘价在前一根最高价**附近** | `open ≥ prev.high − prev.range × 0.3` | 开在前一根振幅的**上 30%** 区间内（或更高/跳空），与绝对价位无关。**射击之星、上吊线**均用此口径 |
| 开盘价在前一根最低价**附近** | `open ≤ prev.low + prev.range × 0.3` | 上一行的**镜像**，开在前一根振幅的**下 30%**（或更低/跳空低开）。**锤子线**用此口径 |
| **很短**的上/下影线 | `影线 ≤ range × 0.15` | 占振幅 15% 以内 |
| 黄昏之星/早晨之星**实体很小** | `body ≤ range × 0.3` | 自参照，允许 body=0（开收相等）。两形态共用 |
| 黄昏之星/早晨之星影线**较短** | `upper/lower ≤ pRange × 0.4` | 锚前一根振幅（"短"是相对趋势K线的尺度）。两形态共用 |
| 黄昏之星/早晨之星影线**相差不大** | `|upper-lower| ≤ range × 0.2` | 自参照，上下影基本等长。两形态共用 |
| 上吊线**小实体** | `body ≤ range × 0.2` | 书把"小实体"列为独立识别标志；仅靠下影≥2倍实体只能隐式压到≤1/3，故显式收到 1/5。只加在上吊线分支，锤子线未动 |
| 长腿车夫**小实体** | `body ≤ range × 0.15` | 允许近似十字，但要求整根K线有足够长影 |
| 长腿车夫**长影线**（长腿） | `upper/lower ≥ max(body × 2, pRange × 0.5)` | "2倍实体"是书面数值；长腿锚**前一根振幅**（绝对尺度），与黄昏之星"短影 ≤ pRange×0.4"共用同一把尺子 |
| 墓碑线**大幅高开** | `gapPct ≥ 0.07` | "接近涨停"的量化；偏松，可收紧到约 0.09 |
| 墓碑线**振幅** | `rangePct ≥ 0.07` | 书面数值"振幅≥7%"（`range/prev.close`）|
| 墓碑线**无/很短上下影** | `upper/lower ≤ range × 0.1` | 光头光脚大阴线；body≥80% 由此隐含 |
| 中/长实体 | `body ≥ range × 0.4` | 用于乌云盖顶的中阳/长阳、中阴/长阴 |
| 收在最低价附近 | `lower ≤ range × 0.15` | 用于乌云盖顶第二根阴线 |
| 中小阴线 | `down && range×0.25 ≤ body ≤ range×0.75` | 用于三只乌鸦，排除十字和超长实体 |
| **上升/下跌波段** | `trend`：4 根、收盘涨跌幅 ±1% | 见 §2 |

> 影线"≥实体2倍"、"前一根最高价"这类**有明确数值/对象**的条件，按书面原话写死，不在旋钮内。

## 4. 形态逐条对照

### 4.1 射击之星 ✅ 已核对（顶部反转 / 看跌，标记「射」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价在前一根K线的最高价附近；
2. 上影线长度是实体的2倍以上；
3. 没有下影线或只有很短的下影线。

> 颜色不重要（绿阴更有效）；实体越短越看跌；通常伴随放量。

**代码条件**（全部满足才标记）：
```js
i >= 1 && f.body > 0 &&
f.upper >= f.body * 2 &&                              // ② 上影 ≥ 实体2倍
f.lower <= f.range * 0.15 &&                          // ③ 无/很短下影
c.open  >= rows[i-1].high - rows[i-1].range * 0.3 &&  // ①b 开在前一根振幅上30%
trend(i - 1) > 0                                      // ①a 上升波段顶部
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘价在前一根最高价附近 | `open ≥ prev.high − prev.range × 0.3` | ✅ |
| ② 上影线 ≥ 实体2倍 | `upper ≥ body × 2` | ✅ |
| ③ 无/很短下影 | `lower ≤ range × 0.15` | ✅ |
| 颜色不限、实体越短越看跌 | `body > 0`，不设实体下限 | ✅ |

**取舍说明：** 书面①b 只说"开盘价在前一根最高价**附近**"，没给数值。"附近"的口径由实现量化——旧版取 `≥ prev.high × 0.99`（容差仅 1%），等价于要求**前一根必须收在自己最高价的 1% 以内**，前一根一旦有正常上影，"开在前收"就会被卡掉（卡掉它的是这 1%，不是书）。现改为 **`open ≥ prev.high − prev.range × 0.3`**（开在前一根振幅的上 30% 区间），与绝对价位无关，对大阳线/带上影的前一根都稳，更贴"附近"本意。仍是**单侧下限**：跳空高开不排除（书中"跳空越大越看跌"）。

### 4.2 上吊线 ✅ 已核对（顶部反转 / 看跌，标记「吊」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价在前一根K线最高价附近；
2. 没有上影线或只有很短的上影线；
3. 下影线长度是实体的2倍以上；
4. 小的实体，实体颜色不很重要。

> 收阴线（绿实体）表明收盘不能反弹至开盘价，卖方动能在聚集。

**代码条件：**
```js
// 共同形状（②③，与锤子线共用）
f.body > 0 && f.lower >= f.body * 2 && f.upper <= f.range * 0.15
// 上吊线（在上述形状基础上，含④小实体显式门槛）
&& i >= 1 && trend(i - 1) > 0
&& f.body <= f.range * 0.2
&& c.open >= rows[i-1].high - rows[i-1].range * 0.3
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘价在前一根最高价附近 | `open ≥ prev.high − prev.range × 0.3` | ✅ |
| ② 无/很短上影 | `upper ≤ range × 0.15` | ✅ |
| ③ 下影线 ≥ 实体2倍 | `lower ≥ body × 2` | ✅ |
| ④ 小的实体、颜色不重要 | `body ≤ range × 0.2`，不要求阴阳 | ✅ |

**①b 口径（同射击之星）：** 书面只说"开盘价在前一根最高价**附近**"，无数值。"附近"量化为 `open ≥ prev.high − prev.range × 0.3`（开在前一根振幅的上 30% 区间，或更高/跳空）。书中上吊线的两个实例（图3-6 宏润建设、图3-7 翼中能源）都是**高开低走**，跳空高开天然满足此条件。旧版 `× 0.99` 已弃用。

**④小实体口径：** 书把"小实体"列为独立识别标志（开篇"以小实体、短上影、长下影作为识别标志"，排在第一位），但书未给数值。仅靠 ②③ 隐式只能把实体压到 **≤ 振幅 1/3（≈33%，上影为 0 时取到上确界）**——1/3 算不上"小"，故显式补 `body ≤ range × 0.2`（实体 ≤ 振幅 1/5）。代价：实体在 20%~33% 且带长下影的 K 线虽满足书中唯一数值条件 ③，仍会被这条更严的口径挡掉——这是把定性词"小"量化的必然取舍。
（旧文档把隐式上界写成"约 28%"是错的：28% 只是「上影取满 15%」那一刻的实体占比，并非最大值。）
**注：** 此 ④ 门槛**只加在上吊线分支**；共用形状门槛（②③）未动。锤子线书面特征列表**未单列"小实体"**（见 §4.3），故沿用 ②③ 隐含的 ≤1/3。

### 4.3 锤子线 ✅ 已核对（底部反转 / 看涨，标记「锤」）

**书面形态特征：**
1. 位于下降波段底部，开盘价在前一根 K 线的最低价附近；
2. 下影线长度是实体的 2 倍以上；
3. 没有上影线或只有很短的上影线；
4. 实体颜色不重要。

与上吊线**形状完全相同**（长下影、小实体、短上影），共用形状门槛 `body>0 && lower≥body×2 && upper≤range×0.15`，靠**趋势 + 开盘位置**区分。

**代码条件**（在共用形状基础上；`pRange = prev.high − prev.low`）：
```js
trend(i - 1) < 0 &&                       // ① 下降波段底部
c.open <= prev.low + pRange * 0.3         // ① 开盘在前一根最低价附近（前一根振幅下30%，含跳空低开）
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 下降波段底部 | `trend(i-1) < 0` | ✅ |
| ① 开盘在前一根最低价附近 | `open ≤ prev.low + pRange × 0.3`（上吊线"前高附近"的镜像） | ✅ |
| ② 下影线 ≥ 实体2倍 | `lower ≥ body × 2` | ✅ |
| ③ 无/很短上影 | `upper ≤ range × 0.15` | ✅ |
| ④ 颜色不重要 | 不要求阴阳 | ✅ |

**①b 口径：** 旧版锤子线只有 `trend(i-1) < 0`、**没有开盘位置约束**——会把"同样长下影小实体但高开"的 K 线也误标为锤子线。现按书补上「开盘在前一根最低价附近」，取上吊线"前高附近"的**镜像** `open ≤ prev.low + pRange × 0.3`。

**④小实体（与上吊线的差异）：** 书面**锤头线**的特征列表里**没有"小实体"这一条**（只有"实体颜色不重要"），而**上吊线**列表明确写了"小的实体"。因此锤子线**不加** `body ≤ range×0.2` 的显式门槛，沿用 ②③ 隐含的 ≤振幅1/3。若想让两个镜像形态口径一致，可单独加，但那已超出锤头线的书面特征列表。

### 4.4 黄昏之星 ✅ 已核对（顶部反转 / 看跌，标记「昏」）

**书面形态特征：**
1. 位于上升波段顶部，开盘价通常在前日收盘价之上；
2. 上影线与下影线较短，且长度相差不大；
3. 实体很小或开盘价等于收盘价；
4. 实体颜色不重要。

**代码条件**（全部满足才标记，标在小星本身这根K线上；`pRange = prev.high − prev.low`）：
```js
i >= 1 &&
trend(i - 1) > 0 &&
open >= prev.close &&                  // ① 开盘在前日收盘价之上
body <= range * 0.3 &&                 // ③ 实体很小（允许 body=0，开收相等）
upper <= pRange * 0.4 &&               // ② 上影较短（相对前一根振幅）
lower <= pRange * 0.4 &&               // ② 下影较短
Math.abs(upper - lower) <= range * 0.2 // ② 上下影基本等长（自参照）
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ①a 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ①b 开盘通常在前日收盘价之上 | `open ≥ prev.close` | ✅ |
| ② 上下影线较短 | `upper/lower ≤ pRange × 0.4` | ✅ |
| ② 上下影线长度相差不大 | `|upper-lower| ≤ range × 0.2`（自参照） | ✅ |
| ③ 实体很小或开收相等 | `body ≤ range × 0.3`，允许 `body = 0`（自参照） | ✅ |
| ④ 实体颜色不重要 | 不要求阳线/阴线 | ✅ |

**取舍说明：**
1. 旧实现是"阳线 → 小星 → 阴线跌破首根实体中点"的三根确认版。现已改为书面定义中的**单根小星本体**，确认阴线不参与是否标记。
2. **锚点（重点）：** 书面 ②③ 描述的是**星这根自己**的形状，书里只有 ① 引用前一根（且用其**收盘价**）。旧版把实体、上下影、对称性**四样全锚到前一根实体 `prev.body`**，导致两类错：(A) 前一根实体偏小时，把教科书式小对称星**漏标**；(B) 前一根实体大时，把"短上影+无下影"这种**明显不对称**的星错标成黄昏之星（违反书面"基本等长"）。现按书改为：**③实体、②对称性自参照**（锚星自身振幅），仅 **②"较短"** 因"短/长"是相对趋势K线的尺度概念而锚 **前一根振幅 `pRange`**。
3. **与长腿车夫的边界（已统一）：** "短影黄昏之星"与"长影长腿车夫"本质是同一种小对称星，靠**影线长短**区分。两者现已共用同一把尺子 `pRange`：**影线 ≤ pRange×0.4 → 昏**、**影线 ≥ pRange×0.5 → 车**，中间为模糊带不标，二者不再重叠。书面"开收相等"的纯十字短腿黄昏之星已能正确标「昏」（长腿车夫的"长腿"门槛也已从自参照 `range×0.3` 改为 `pRange×0.5`，见 §4.5）。

### 4.5 长腿车夫 ✅ 已核对（顶部反转 / 看跌，标记「车」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 上影线与下影线较长，均长于实体的2倍；
3. 实体较小，实体颜色不重要。

**代码条件**（全部满足才标记；`pRange = prev.high − prev.low`）：
```js
i >= 1 &&
trend(i - 1) > 0 &&
body <= range * 0.15 &&                  // ③ 实体较小
upper >= Math.max(body * 2, pRange * 0.5) && // ② 上影 ≥2倍实体 且 长腿(≥前一根振幅一半)
lower >= Math.max(body * 2, pRange * 0.5)    // ② 下影同上
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 上影线 ≥ 实体2倍 | `upper ≥ body × 2`（书面数值） | ✅ |
| ② 下影线 ≥ 实体2倍 | `lower ≥ body × 2`（书面数值） | ✅ |
| ② 上下影线都很长（"震荡幅度相当大"） | `upper/lower ≥ pRange × 0.5` | ✅ |
| ③ 实体较小、颜色不重要 | `body ≤ range × 0.15`，不要求阴阳 | ✅ |

**取舍说明（锚点修正）：** 旧版用 `≥ range × 0.3`（占**自身**振幅 30%）来表示"长腿"——但这是**尺度盲点**：一根绝对很小的迷你十字，影线照样占自身振幅一半 ≥ 30%，会被错当长腿车夫；而书面强调的是"上下影**都很长**、当日**震荡幅度相当大**"，是**绝对/相对趋势**的长度。故改为锚**前一根振幅** `pRange × 0.5`（影线至少为前一根整根振幅的一半），书面"≥2倍实体"作为另一道并存门槛保留。

**与黄昏之星的边界（已统一）：** 黄昏之星与长腿车夫本是同一种"小实体对称星"，只靠**影线长短**分。现在两者用同一把尺子（`pRange`）：**影线 ≤ pRange×0.4 → 短 → 昏**；**影线 ≥ pRange×0.5 → 长 → 车**；中间 (0.4, 0.5) 为模糊带，不标。二者**不再重叠**，书面"开收相等"的**纯十字短腿黄昏之星不再被长腿车夫抢走**。

### 4.6 墓碑线 ✅ 已核对（顶部反转 / 看跌，标记「墓」）

> **重要：本书的"墓碑线"不是西式 gravestone doji（小实体+长上影）。** 本书定义为"上升波段顶部、接近涨停大幅高开后全天回落、收在最低价附近形成的**很长的绿色阴线（光头光脚大阴线）**，振幅≥7%"。旧文档把它写成 gravestone 十字是**错的**（书面特征和代码块都写错了），现按书 + 实际代码改正。

**书面形态特征：**
1. 位于上升波段顶部；
2. 以接近涨停的价格大幅高开；
3. 没有上下影线或只有很短的下影线；
4. 很长的绿色（阴线）实体，振幅通常在 7% 以上。

**代码条件**（全部满足才标记；`gapPct=(open−prev.close)/prev.close`、`rangePct=range/prev.close`）：
```js
i >= 1 &&
trend(i - 1) > 0 &&
down &&                       // ④ 绿色阴线（close < open）
gapPct >= 0.07 &&             // ② 大幅高开（接近涨停）
rangePct >= 0.07 &&           // ④ 振幅 ≥ 7%
body >= range * 0.75 &&       // ④ 很长实体（实为冗余，由下两条上下影≤10%已隐含 body≥80%）
upper <= range * 0.1 &&       // ③ 无/很短上影
lower <= range * 0.1          // ③ 无/很短下影
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 接近涨停大幅高开 | `gapPct ≥ 0.07` | ✅（"接近涨停"约+10%，7% 是偏松的下限，见取舍） |
| ③ 无/很短上下影 | `upper ≤ range×0.1 && lower ≤ range×0.1` | ✅ |
| ④ 很长绿色阴线实体 | `down && body ≥ range×0.75` | ✅ |
| ④ 振幅 ≥ 7% | `rangePct ≥ 0.07` | ✅（书面数值） |

**与书面实例核对：** 图3-13 中信证券（高开 8.77%/振幅 7.37%/收前收之上但仍阴线）与图3-14 中国联通（+10% 涨停开盘/收 −1.57%/振幅 12.88%）实跑均**正确标「墓」**；"大幅高开但收阳线""高开仅 4%"等反例正确不标。

**取舍说明：**
1. **`gapPct ≥ 0.07`** 是把"接近涨停大幅高开"量化的旋钮。书面与两个实例都接近 +9%~+10%（涨停区），7% 偏松——若想更贴"接近涨停"，可收紧到约 0.09。另外此处用固定 7%，未区分主板(±10%)/创业板科创板(±20%)的涨停幅度。
2. **`body >= range × 0.75` 是冗余条件**：由 `upper ≤ range×0.1` 与 `lower ≤ range×0.1` 已推出 `body ≥ range×0.8`，恒成立。保留它仅作"很长实体"的意图说明，可删。

### 4.7 乌云盖顶 ✅ 已核对（顶部反转 / 看跌，标记「乌」）

**书面形态特征：**
1. 位于上升波段顶部；
2. 第一根K线为一根中阳或长阳线；
3. 第二根K线为高开，并收盘在最低价附近的中阴或长阴线；
4. 第二根K线的收盘价在第一根K线实体的半分位以下。

**代码条件**（全部满足才标记）：
```js
i >= 1 &&
prev.up &&
prev.body >= prev.range * 0.4 &&
down &&
body >= range * 0.4 &&
lower <= range * 0.15 &&
trend(i - 1) > 0 &&
open > prev.close &&
close < (prev.open + prev.close) / 2 &&
close > prev.open
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 上升波段顶部 | `trend(i-1) > 0` | ✅ |
| ② 第一根中阳/长阳 | `prev.up && prev.body ≥ prev.range × 0.4` | ✅ |
| ③ 第二根高开 | `open > prev.close` | ✅ |
| ③ 第二根中阴/长阴 | `down && body ≥ range × 0.4` | ✅ |
| ③ 第二根收在最低价附近 | `lower ≤ range × 0.15` | ✅ |
| ④ 第二根收盘低于前实体半分位 | `close < (prev.open + prev.close)/2` | ✅ |

**取舍说明：** 代码保留 `close > prev.open`，即第二根没有完全吞没第一根；如果收在第一根开盘价以下，会优先归类为看跌吞没。

### 4.10 早晨之星（启明星）✅ 已核对（底部反转 / 看涨，标记「晨」）

书面明说早晨之星是"**与顶部黄昏形态相对应的底部形态**"，即 §4.4 黄昏之星的**镜像**。除趋势与开盘方向外，形状条件（小实体、上下影较短且基本等长）与黄昏之星**完全相同**。

**书面形态特征：**
1. 位于下降波段底部，开盘价在前日收盘价之下；
2. 上影线与下影线较短，且长度相差不大；
3. 实体很小或开盘价等于收盘价；
4. 实体颜色不重要。

**代码条件**（`pRange = prev.high − prev.low`）：
```js
i >= 1 &&
trend(i - 1) < 0 &&                    // ① 下降波段底部
open <= prev.close &&                  // ① 开盘在前日收盘价之下（低开）
body <= range * 0.3 &&                 // ③ 实体很小（允许 body=0）
upper <= pRange * 0.4 &&               // ② 上影较短
lower <= pRange * 0.4 &&               // ② 下影较短
Math.abs(upper - lower) <= range * 0.2 // ② 上下影基本等长（自参照）
```

| 书面特征 | 代码 | 状态 |
|---|---|---|
| ① 下降波段底部 | `trend(i-1) < 0` | ✅ |
| ① 开盘在前日收盘价之下 | `open ≤ prev.close`（黄昏之星 `≥` 的镜像） | ✅ |
| ② 上下影较短 | `upper/lower ≤ pRange × 0.4` | ✅ |
| ② 上下影长度相差不大 | `|upper-lower| ≤ range × 0.2`（自参照） | ✅ |
| ③ 实体很小或开收相等 | `body ≤ range × 0.3`，允许 `body = 0` | ✅ |
| ④ 颜色不重要 | 不要求阴阳 | ✅ |

**说明：** 此前代码**未实现**早晨之星，本次按黄昏之星镜像补上。与锤子线（也是底部、低开的小实体长下影）互斥：早晨之星要求上下影**短而对称**，锤子线要求**长下影**，实跑不会重复命中。底部没有"长腿车夫"对应形态，故不存在黄昏/长腿那种短长边界耦合。

## 6. 未纳入硬判定的部分（属"确认/强度"修饰，不是形态特征本身）

书里这些是**确认信号**或**有效性修饰**，当前**不参与**是否标记的判断：

- **确认信号**（需下一根K线）：下一根开盘在形态实体下方 / 下一根阴线收盘低于形态收盘价。
- **放量**：成交量越大，冲顶/见顶概率越高。
- **颜色权重**：绿色（阴线）的射击之星/上吊线更有效。

如需把"确认信号"做成更严格的"已确认"模式，或把放量/颜色作为强度加权，再单独议。
